## 1. 評価概要

### 1.1 評価目的

Coro Cybersecurityプラットフォームのv3.7およびv3.8でリリースされた新機能・機能強化について、以下の観点から包括的な評価を実施する。

1. **機能の完全性**: リリースノートに記載された機能が正しく実装されているか
2. **セキュリティ効果**: 新機能がセキュリティ保護にどの程度貢献するか
3. **運用性**: 管理者・ユーザーの運用負荷への影響
4. **統合性**: 既存機能・外部システムとの連携
5. **信頼性**: 機能の安定性とエラー処理

### 1.2 評価範囲

```

│                        評価対象範囲                              │
```

- パフォーマンス負荷テスト
- ペネトレーションテスト

## 3. 詳細評価項目

### 3.1 セキュリティ機能評価

#### 3.1.1 AI/ML関連脅威検知

| 評価ID | 評価項目 | 対象バージョン | 優先度 | 評価内容 |
|--------|----------|----------------|--------|----------|
| SEC-AI-001 | Prompt Injection Detection | v3.7 | **最高** | メール本文・件名内のAIプロンプトインジェクション検知精度 |
| SEC-AI-002 | Shadow AI Blocklist | v3.7 | 高 | AIチャットボットへのアクセスブロック機能 |
| SEC-AI-003 | Coro AI Summary | v3.7/v3.8 | 中 | AI支援によるセキュリティサマリー精度 |

**SEC-AI-001 詳細評価手順**

```
1. 準備
   - テスト用メールアカウント設定
   - Email Security有効化確認

2. テストケース
   TC-001: 明示的プロンプトインジェクション
   - 件名: "Ignore previous instructions and..."
   - 期待: 検知・警告/隔離

   TC-002: 隠蔽されたプロンプト（Base64等）
   - 本文にエンコードされたプロンプト
   - 期待: 検知可否の確認

   TC-003: 正常メールの誤検知確認
   - AI関連の正当な業務メール
   - 期待: 正常通過

3. 評価観点
   - 検知率（True Positive Rate）
   - 誤検知率（False Positive Rate）
   - 検知までの時間
   - 管理者通知の適切性
```

#### 3.1.2 認証・アクセス制御

| 評価ID | 評価項目 | 対象バージョン | 優先度 | 評価内容 |
|--------|----------|----------------|--------|----------|
| SEC-AUTH-001 | MFA Status Detection | v3.8 | **最高** | Microsoft 365/Google WorkspaceのMFA状態検出精度 |
| SEC-AUTH-002 | Threat Detection Policies | v3.7 | 高 | 新しい脅威検知タイプの動作確認 |

**SEC-AUTH-001 詳細評価手順**

```
1. 前提条件
   - Microsoft 365連携設定済み
   - UserAuthenticationMethod.Read.All権限付与済み
   - Google Workspace連携設定済み

2. テストケース
   TC-001: MFA有効ユーザーの検出
   - MFA設定済みユーザーでログイン
   - Protected Usersページで"MFA enabled"ラベル確認

   TC-002: MFA無効ユーザーの検出
   - MFA未設定ユーザー
   - "MFA not enabled"ラベル確認

   TC-003: MFA状態変更の反映
   - MFA状態を変更後、反映タイミング確認

   TC-004: フィルタリング機能
   - MFA状態によるユーザーフィルタ動作確認

3. 評価観点
   - 検出精度
   - 状態更新の遅延時間
   - UI表示の正確性
   - フィルタ機能の正常動作
```

#### 3.1.3 脅威検知ポリシー（v3.7 Cloud Security）

| 検知タイプ | 評価項目 | テスト方法 |
|------------|----------|------------|
| Abnormal Admin Activity | 異常な管理操作の検知 | 通常時間外の管理操作シミュレーション |
| Mass Data Deletion | 大量データ削除の検知 | 閾値超過の削除操作テスト |
| Mass Data Download | 大量ダウンロードの検知 | 閾値超過のダウンロードテスト |
| Suspected Bot Attack | ボット攻撃の検知 | 自動化パターンのシミュレーション |
| Suspected Identity Compromise | ID侵害の検知 | 異常ログインパターンのシミュレーション |

## 6. 評価基準

### 6.1 評価スコアリング

| スコア | 評価 | 基準 |
|--------|------|------|
| 5 | 優秀 | 期待以上の機能・性能 |
| 4 | 良好 | 期待通りの機能・性能 |
| 3 | 許容 | 軽微な問題はあるが使用可能 |
| 2 | 要改善 | 重要な問題があり改善が必要 |
| 1 | 不可 | 機能せず使用不可 |

### 6.2 総合評価基準

| 評価 | 条件 |
|------|------|
| **推奨** | 全最高優先度項目が4以上、かつ平均スコア3.5以上 |
| **条件付き推奨** | 最高優先度項目に3があるが、重大な問題なし |
| **保留** | 最高優先度項目に2以下があり、改善待ち |
| **非推奨** | 複数の重大な問題があり、導入リスクが高い |
